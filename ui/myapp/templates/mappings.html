Ordered queue timing

Controls event settling, ownership recovery, and safe continuation decisions. Changes affect new worker invocations globally.

Configuration notes
  • Writer workers are threads inside one Function invocation; provisioned concurrency controls the number of pre-warmed Function instances.
  • Higher concurrency can increase simultaneous database writes and Object Storage reads. Validate HeatWave connections, IOPS, redo capacity, and tenancy limits before increasing it.
  • Reorder grace intentionally adds waiting time. Runtime reserves and admission estimates decide whether another entry starts or is handed to a continuation.
  • Updating the Function configuration is global and affects every mapping and rule that invokes this Function.
