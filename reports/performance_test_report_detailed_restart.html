2. Data and execution method

Files were generated on the VM with deterministic rows using the target schema:

record_id,event_ts,category,payload,amount

event_ts uses MySQL DATETIME syntax (YYYY-MM-DD HH:MM:SS). Objects were uploaded with OCI CLI. The Function streamed Object Storage ranges directly; no temporary CSV was created inside the Function container.

3. Timing definitions

  • Upload duration: wall-clock interval around oci os object put on the VM.
  • Function duration: OCI invocation log interval from “Received function invocation request” to “Served function invocation request”.
  • End-to-end latency: upload completion to terminal Event TX success; this requires correlating the upload timestamp with the control-schema terminal record.

6. Long-running execution limitation

Object Storage Events invoke the FAAS action synchronously. The Function cannot run beyond OCI's timeoutInSeconds=300 limit; the 1 GB test reached FunctionInvokeTimeout (504) at approximately 300 seconds.

Detached invocation supports up to 3,600 seconds, but is not selected directly by the current Event Rule. A future design needs a short Event Rule handler that persists the event and submits a detached worker, or routes through Queue/Streaming. The worker carries object identity, ETag, mapping ID, and correlation ID, with states RECEIVED → DETACHED_SUBMITTED → PROCESSING → SUCCESS/ERROR.

7. Observations

  • The 500 MB run completed in approximately four minutes and used 32 MiB Object Storage range responses.
  • Small files are dominated by fixed Event Rule/Function startup overhead; larger files expose streaming and database load throughput.
  • The 1 GB case is intentionally detached because generation and upload exceed an interactive terminal window.
  • Do not project p95/p99 or sustainable throughput from this sequential sample; concurrency and repeated trials remain outstanding.
Next steps: wait for the 1 GB marker, collect its Function log, then run fan-out (10–100 small files) and mixed create/update/delete cases. Capture terminal Event TX timestamps and database row counts for the final capacity report.
