Measurement contract

Each object records a UTC upload-start and upload-complete timestamp. The Event TX and Object Storage Event pages provide received, completed, and duration values. End-to-end latency is terminal completion minus upload-complete. Results must include bytes, rows, status, retries, and error text.

Execution note: the new VM deployment and isolated Event Rule were verified ACTIVE. The VM does not have the MySQL CLI installed, so database-side test-table/mapping setup must use the application’s Connector/Python environment or the UI before collecting measured rows. This report intentionally does not invent timing values.

Acceptance criteria

  • No Function temporary CSV file is created.
  • Every uploaded object has exactly one terminal lifecycle result.
  • Create/update use staging and partition exchange; delete retires the corresponding batch.
  • Report includes p50/p95/p99 latency, MB/s, objects/minute, and error rate.
